# 报告输出模板

以下为完整报告的结构和格式要求。输出时替换所有占位符为实际内容。

---

## 🛠️ 今日工具探索报告 - {当前日期}

---

### 📱 实用 App & 小工具推荐（Top 5-8）

每个工具使用以下固定格式：

```
#### 🔧 工具名称（附访问链接）
> 一句话定位：用最简洁的语言说明这个工具解决什么问题

- **解决的痛点**：描述用户在没有这个工具时会遇到什么具体困扰，2-3 句话展开
- **核心功能**：① [功能1] ② [功能2] ③ [功能3]
- **适合人群**：[目标用户]
- **平台**：iOS / Android / Web / 桌面 / CLI / 浏览器插件
- **价格**：免费 / 买断（注明价格）/ 订阅（注明价格区间）
- **信息来源**：Product Hunt / GitHub / 少数派 / 小红书 / 其他
```

---

### 📦 GitHub 工具类小项目（Top 3，仅限工具类）

如果 GitHub Trending 无合适工具类项目，此部分可减少数量或标注"本期 GitHub Trending 以框架/平台类项目为主，工具类项目较少"。

每个项目包含：

```
#### ⭐ 项目名称（GitHub 链接）
> 一句话定位：这个项目解决什么日常问题

- **使用场景**：描述一个具体的使用场景（2 句话，让人有代入感）
- **目标用户**：[具体用户画像]
- ⭐ Star 数：[数量]（近期趋势：上升/稳定/新项目）
- **技术栈**：[主要技术]
```

---

### 💡 产品 Ideas & 拓展方向（Top 5）

这是报告的核心价值部分。每个 Idea 基于前述工具分析，给出可拓展的新方向。

```
#### 💡 Idea [序号]：[Idea 名称]

- **核心场景**：[2-3 句话描述真实的日常使用场景，要让人有共鸣]
- **目标用户群体**：[具体到人群画像]
- **差异化切入点**：[和现有工具相比的独特之处]
- **灵感来源**：[基于前述哪个/哪些工具的分析得出]
- **潜在商业模式**：[订阅/买断/免费增值/广告，注明预期定价]
- **难度评估**：[⭐~⭐⭐⭐⭐⭐]
```
